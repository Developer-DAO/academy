### **Blockchains, briefly**

There are many ways to describe Ethereum, but at its heart is a blockchain. Blockchains are made up of a series of blocks, so letâ€™s start there. In the simplest terms, each block on the Ethereum blockchain is some metadata and a list of transactions. In JSON format, that looks something like this:

```python

   "hash": "0xabc123...",

   "transactions": [...]
```

Each block has a reference to the block that came before it; theÂ `parentHash`Â is simply the hash of the previous block.

A blockchain is essentially a linked list; each block has a reference to the previousÂ block.

This data structure is nothing novel, but the rules (i.e., peer-to-peer protocols) that govern the network are. Thereâ€™s no central authority; the network of peers must collaborate to sustain the network, and compete to decide which transactions to include in the next block. So, when you want to send some asset to a friend, youâ€™ll need to broadcast that transaction to the network, then wait for it to be included in an upcoming block.

In Ethereum, this native currency is calledÂ **ether**, and the Ethereum blockchain contains the only official record of account balances.
